Paramètres du reste et opérateur de décomposition des fonctions JavaScript

Souvent, dans le cas d’opérations mathématiques, on aimerait faire en sorte qu’une fonction puisse recevoir un nombre variable d’arguments.

Les paramètres du reste ...

... : permet de définir un nombre variable d'arguments dans une fonction.

Lorsqu’on déclare des paramètres de cette manière, on parle de « paramètres du reste ». Littéralement, on demande au JavaScript de stocker les arguments passés dans un tableau qui possèdera le nom mentionné après ….

On va également pouvoir créer des fonctions en précisant des paramètres de manière « classique » et en utilisant aussi des paramètres du reste.
Dans ce cas-là, seuls les arguments supplémentaires passés à la fonction seront stockés dans le tableau.
Attention cependant : il faudra toujours préciser les paramètres classiques en premier et les paramètres du reste en dernier dans la déclaration de la fonction.

Mettons en pratique :

let nom = "Banner", prenom = "David";
let p1 = document.getElementById('p1');

function profil(nom, prenom, ...hobbies ) {
let h = '';
// on boucle sur le tableau hobbies
for(hobbie of hobbies){ h += hobbie + ' - '; }
return nom + ' ' + prenom + ' : ' + h;
}

L’opérateur de décomposition

Les paramètres du reste permettent de stocker une liste d’arguments dans un tableau qu’on va ensuite pouvoir manipuler.

L’opérateur de décomposition permet de faire l’opération inverse, à savoir transformer un tableau en une liste d’arguments qu’on va pouvoir passer à une fonction.

L’opérateur de décomposition utilise la même syntaxe avec … que les paramètres du reste à la différence qu’on va faire suivre les trois points par le nom d’un tableau existant.

En pratique :

Servons-nous de la fonction js Math.max() :

Math.max(5, 10, 20, 15); - // renvoie 20

let tb1 = [1, 2, 3, 4];
Math.max( tb1 ); - // retourne NaN , car reçoit un tableau en argument
math.max( ...tb1 ); - // 4 . L'opérateur de décomposition transforme le tableau en ligne

Cette façon de faire va être très utile dans le cas où on ne sait pas à l’avance combien de nombres on va passer à max.
