Le mode strict en JavaScript

Le mode strict de JavaScript est, comme son nom l’indique, un mode qui va contenir une sémantique< légèrement différente et plus stricte par rapport au JavaScript « classique ».
Le mode strict va notamment lever plus d’erreurs que le JavaScript classique.

Activer et utiliser le mode strict

Pour activer le mode strict, nous allons utiliser la syntaxe use strict . On va pouvoir choisir d’activer le mode strict

  • use strict : dans tout le script, ajoutez l'instruction au tout début de celui-ci.

    use strict;

    // le reste du code vient ici

  • use strict : dans une fonction uniquement, ajoutez use strict dans le code de la fonction, au tout début.

Attention : si l’instruction use strict n’est pas la première dans le script ou dans une fonction, alors le mode strict ne fonctionnera pas.

Le contenu des modules JavaScript est automatiquement en mode strict.

Exemples de code qui vont déclencher une erreur en mode strict (et pas en mode classique :

  • Déclarer une variable sans let ou var
  • Affecter une valeur à une variable ou une propriété non accessible en écriture;
  • Tenter de supprimer une propriété non supprimable (comme la propriété prototype par exemple);
  • Déclarer plusieurs paramètres de même nom dans une fonction;
  • Utiliser des notations octales (si on fait précéder un nombre par un 0);
  • Définir des propriétés sur des valeurs primitives;
  • Utiliser with et eval() (fortement déconseillé, même en mode classique;
  • Utiliser this.
    La valeur de this en mode strict n’est pas transformée en objet (contrairement au mode classique) et si elle n’est pas définie la valeur passée sera undefined.
  • utiliser arguments;
    La propriété arguments, quant-à-elle, ne permet pas d’accéder aux variables passées à la fonction lors de son appel en mode strict. Cela permet également de réduire les failles de sécurité.
  • Le mode strict et les noms réservés.
    En mode strict, il est interdit d’utiliser les termes implements, interface, let, package, private, protected, public, static, et yield pour nommer des variables ou des arguments car ce sont des mots réservés.
